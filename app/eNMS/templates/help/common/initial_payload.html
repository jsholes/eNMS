The initial_payload parameter provides starting data that can be referenced later by service(s). This parameter is a dictionary with key/value pairs. This parameter is always present and will default to "{ }".

Important note: only one initial_payload matters during a run: the one populated for the service or workflow being run. If the user runs a workflow, the initial_payload defined on the workflow will be used (and any service initial_payload ignored). If the user runs an individual service, that service's initial_payload will be used if defined (and the workflow initial_payload ignored if defined).
